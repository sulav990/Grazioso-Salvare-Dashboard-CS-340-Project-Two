# Grazioso-Salvare-Dashboard-CS-340-Project-Two
Grazioso Salvare Dashboard – CS 340 Project Two
Writing maintainable, readable, and adaptable programs comes down to keeping things modular and well documented. For the Grazioso Salvare project, building the CRUD Python module first was a perfect example of this. Instead of cluttering the dashboard code with direct database connections and raw queries, I separated the data layer from the user interface. This separation of concerns made the dashboard application much easier to read and debug. The biggest advantage of working this way is reusability. Because the CRUD operations are contained in their own class, I can easily take this exact module, plug it into a completely different Python application, and instantly have a secure way to interact with a MongoDB database without having to rewrite the core logic.

When approaching a problem as a computer scientist, my first step is always to deeply understand the end goal of the user. For the Grazioso Salvare dashboard, this meant looking closely at the specific dog breeds, age ranges, and training requirements the client needed before writing any code. I had to ensure the database could actually support those specific queries. This project felt much more grounded in reality compared to previous assignments because it involved taking a massive, messy dataset from the Austin Animal Center and turning it into a polished, client facing product rather than just solving an abstract algorithmic puzzle. In the future, I will continue to use this strategy of mapping out the entire data pipeline from the raw database to the final user interface before jumping into the actual development phase.

At its core, what computer scientists do is solve complex problems by making information accessible and actionable. It matters because we build tools that allow people to focus on their actual mission rather than getting bogged down by tedious manual processes. For a company like Grazioso Salvare, manually sorting through thousands of shelter records to find a dog matching highly specific search and rescue criteria would take hours or even days. By delivering a dynamic, easily filterable dashboard, the application automates that heavy lifting. This allows the trainers to quickly identify the perfect rescue dog candidates, saving the organization valuable time and ultimately helping them save more lives in the field.
